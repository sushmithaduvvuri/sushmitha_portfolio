Secure File Sharing (Hybrid Cryptography)

Hybrid Encryption Strategy

A two-stage security process using **AES-256 (GCM mode)** for integrity, paired with a profile-based secondary layer: **Triple DES** for large files and **Blowfish** for small data profiles.

Cloud Fragmentation

Instead of a single blob, data is split into **7 distinct fragments** and uploaded to Firebase via REST API, ensuring zero data exposure even if a cloud node is compromised.

Access Control

Multi-role system (Owner, User, Admin) with **two-step verification**. Secret keys are delivered via automated Email once the data owner approves the access request.
